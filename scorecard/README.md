# 博主成绩单（scorecard）

把博主的「前瞻喊单」用**真实行情**长期记账打分，区分"真有 alpha"和"普涨蹭热度/纯引流"。

## 严谨性设计

| 环节 | 做法 | 为什么严谨 |
| --- | --- | --- |
| **只记前瞻喊单** | 只计入"今日关注/计划/方向/低吸/调仓/看好"等买入意图；剔除"已翻X倍/喜提X板/X涨停了"等**事后炫耀** | 避免把事后吹嘘当成预测 |
| **无未来函数** | 喊单的决策时点 = **发帖时间**，收益只看发帖**之后** | 不存在偷看未来 |
| **明确入场** | 入场价 = **喊单当日收盘**（保守、可复现） | 不假设能盘前埋伏 |
| **基准超额** | 收益减去**中证1000ETF**同期收益 | 剔除大盘/小盘普涨，留下真 alpha |
| **逐条可审计** | 每条喊单连原文一起入 `data/calls.jsonl` | 任何一条都能核对 |
| **不夸大** | 样本数、T+N 是否到期都如实标注；样本<5 标"样本少" | 一周≈噪声，长期才有意义 |

## 模块

| 文件 | 作用 |
| --- | --- |
| `stocklist.py` | 全 A 股名→代码解析器（全名精确 + 别名表 + 全角/拼音规范化，宁缺毋滥） |
| `extract.py` | 从动态正文抽「前瞻买入/卖出」喊单（按离个股最近的关键词定向，剔除事后炫耀） |
| `market.py` | mootdx 真实日 K → 算 T+1/T+3/T+5 收益、基准超额 |
| `score.py` | 主流程：抓动态→抽喊单→行情验证→**累积台账**→出报告 |

## 用法

```bash
pip install mootdx requests pandas      # mootdx 取真实行情（需国内/住宅 IP）
BILI_COOKIE="<你的B站Cookie>" python3 scorecard/score.py [起始日 截止日]
```

- 每次运行：抓动态 → 合并进台账（按 博主+代码+日期+方向 去重，**累积不覆盖**）→ 重评所有喊单（已到期的 T+N 自动补全）→ 重出报告。
- **每天收盘后跑一次**，台账越积越厚，几周后排名才有统计意义。
- 产出：`data/calls.jsonl`（台账，可审计）、`../digests/BLOGGER_SCORECARD.md`（报告）。

## 指标

- **命中率**：T+N 个股绝对收益 > 0 的比例
- **跑赢基准率**：T+N 超额 > 0 的比例
- **平均/中位超额**：相对中证1000的超额收益

> ⚠️ 仅为对公开言论的客观验证，不构成投资建议。荐股博主有利益冲突，短期表现好也可能是运气。
